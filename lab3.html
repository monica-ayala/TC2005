    Preguntas y respuestas HTML

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    El internet es la red de computadoras conectadas entre sí, y la World Wide Web es una colección de páginas a las que puedes acceder al conectarte a internet

  • ¿Cuáles son las partes de un URL?

    El protocolo (como http), subdominio, dominio, TLD, ruta, Parámetro, Etiqueta.

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    GET : solicita un recurso del servidor web
    HEAD : solicita solo el encabezado, sin el archivo
    POST : sirve para enviar al servidor web paquetes grandes de datos, adjuntandose en el encabezado html
    PUT : modifica todo el recurso completo
    PATCH : modifica parte de un recurso
    DELETE : eliminar un recurso

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    Se debe utilizar el método de POST

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    Que la petición fue realizada exitosamente, no ocurrió ningún error.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Primeo definamos que es el error 404 para poder entender mejor -> El error 404 es un código de estado que se envía desde el servidor web al navegador, o sea, al usuario que intentaba entrar en dicha página. Básicamente, lo que indica este error es que se trata de un enlace roto, defectuoso o que ya no existe y que, por lo tanto, no es posible navegar por él. Si el enlace es generado automáticamente por la página, es responsabilidad del desarrollador, si el usuario escribió mal el usuario entonces no es responsabilidad del desarrollador.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Primeo definamos que es el error 500 para poder entender mejor -> El Error 500 - Internal server error, es un código de estado HTTP muy general que significa que algo ha ido mal en el servidor del sitio web, pero el servidor no puede ser más específico sobre cuál es el problema exacto. Esto es completamente responsabilidad del desarrollador, ya que es un error interno

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Que sí funciona pero ya no se le da seguimiento ni mantenimiento, ya no se debería seguir usando apesar de que funcione

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    HTML4 mezclaba la presentación del sitio (con tags como center), por lo que se decidió que HTML se dedicaría exclusivamente a la estructura del contenido y para la presentación se emplearía exclusivamente CSS

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    Tanto en HTML4 como HTML5 existe el tag table, tr (table row), td (table data). Sin embargo en HTML5 se incorporan etiquetas semánticas como th (table header) y tfoot (el footer de la tabla). Esto es una ventaja por que el orden de los tags en el código html no importa, dado a que son etiquetas semánticas.

  • ¿Cuáles son los principales controles de una forma HTML5?

    Se creo un elemento input con más capacidades para poder recolectar de mejor manera los datos, manejar errores y validación de errores de mejor manera. Ejemplos: type="number", type="range", type="date", type="color", type="search". Se creo de igual manera el tag datalist, para que al llenar un formulario se pueda obtener una lista de donde el usuario pueda seleccionar una. Se creo también un elemento placeholder para tener un contenido predeterminado en el campo de entrada de datos.

    Preguntas y respuestas CSS

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    Esto no debería de hacerse normalmente, se debería tener una buena estructura en el CSS, con clases, ids y otros selectores para jerarquizar correctamente los estilos de la página, que eviten la necesidad de utilizar el !important.

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    Al escoger una imagen de fondo en una página de HTML debemos cuidar el tamaño de la imagen, ya que puede que se muestre pixeleada si hacemos que ocupe más espacio en la pantalla del de su tamaño, también debemos prestar atención a las proporciones (largo y ancho) de la imágen para que esta pueda acoplarse correctamente a nuestro sitio web.

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    Px - pixeles - no es una buena unidad de medida ya que dependerá de la resolución de cada pantalla y de la configuración que tenga el usuario en su navegador.
    % - porcentajes - es una buena medida ya que se acopla cuando cambiamos el tamaño de la ventana. Nos puede ayudar a que se mantengan las proporciones incluso en computadoras de distinto tamaño de pantalla, resolución y configuración.
    Pt - puntos - es similar a los pixeles, siendo una medida fija que no escala con tamaño. Son buenos para medios impresos pero no tanto para páginas web, ya que los resultados varían mucho de usuario en usuario según su comutadora, ajustes y navegador.

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    Utilizando una versión minimizada del CSS (donde se eliminan todos los comentarios, espacios en blanco y saltos de linea), podemos lograr que el código corra de manera más óptima, mejorando los tiempos de carga de tu página web
